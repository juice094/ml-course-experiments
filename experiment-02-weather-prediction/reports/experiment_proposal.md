### 各方向详细分析

#### A. 温度单步预测（推荐入门）

**为什么选这个？**
- T 列完整度最高（99.99%），数据质量最好
- 温度具有强烈的周期性和趋势性，适合时间序列模型
- 结果直观，易于可视化验证

**建模方案**：
1. **基准模型**：ARIMA / SARIMA（捕捉季节性）
2. **进阶模型**：LSTM / GRU（捕捉非线性模式）
3. **前沿模型**：Informer / Transformer（长序列预测）

**评估指标**：RMSE, MAE, MAPE

**预期难度**：低。SARIMA  baseline 预计 MAPE 在 15-25%。

---

#### B. 温度多变量预测（推荐进阶）

**为什么选这个？**
- 利用湿度、风速、气压等辅助信息，理论上比单变量更准确
- 能体现"多模态数据融合"的机器学习思想
- 特征工程空间大（可构造温差、露点差等衍生特征）

**建模方案**：
1. **传统 ML**：用滑动窗口构造特征矩阵，XGBoost / LightGBM
2. **深度学习**：多变量 LSTM / Seq2Seq / Transformer

**特征工程建议**：
- 时间特征：hour, day_of_year, month, is_weekend
- 气象特征：T-Td（温度露点差，反映饱和程度）
- 滞后特征：T(t-1), T(t-2), T(t-24)（24h前同时间温度）
- 统计特征：过去 24h 平均温度、温度变化率

**评估指标**：RMSE, MAE, R²

**预期难度**：中。特征工程是关键。

---

#### C. 降水二分类（推荐平衡难度与意义）

**为什么选这个？**
- 有实际应用价值（农业灌溉、出行预警）
- 标签可构造：RRR != "无降水" 即为正样本
- 类别不均衡但可处理（用 class_weight 或 SMOTE）

**标签构造**：
```python
# 正样本（有降水）：RRR 不为 "无降水" 且不为 NaN
# 负样本（无降水）：RRR == "无降水"
# 注意：RRR 缺失值需丢弃或单独处理
```

**类别分布预估**：
- 负样本（无降水）：~56%
- 正样本（有降水）：~44%
- 不均衡程度：中等（可用权重调整）

**建模方案**：
1. **基准**：Logistic Regression
2. **主力**：RandomForest / XGBoost
3. **时序**：LSTM + 注意力机制

**评估指标**：Accuracy, Precision, Recall, F1, ROC-AUC（**优先看 F1 和 AUC**）

**预期难度**：中。特征选择比模型选择更重要。

---

#### D. 天气现象多分类（挑战较大）

**为什么谨慎选择？**
- WW 列中 60% 为 "未观测/无法观测"，有效标签仅约 40%
- 天气类型极多（雨、雪、霾、雾、阵雨等），类别不均衡严重
- 部分类别样本极少，模型难以学习

**如果坚持选择**：
- 建议合并类别：晴、雨、雪、其他
- 仅使用 WW 非空且非 "未观测" 的记录

**预期难度**：高。需要大量数据清洗和类别合并工作。

---

#### E. 极端温度预警（快速出结果）

**为什么选这个？**
- 二分类问题，标签构造简单
- 可用传统 ML 快速实现，无需 GPU
- 结果易于解释（特征重要性分析）

**标签构造**：
```python
# 极端高温：T > 20°C（敦煌夏季极端高温阈值）
# 极端低温：T < -20°C（敦煌冬季极端低温阈值）
# 或综合：是否出现极端温度（|T - 月均温| > 2σ）
```

**建模方案**：
1. **基准**：Logistic Regression
2. **主力**：RandomForest / XGBoost
3. **解释性**：SHAP 分析特征贡献

**评估指标**：Precision, Recall, F1（极端事件预测更关注 Recall）

**预期难度**：低。1-2 小时可完成 baseline。

---

#### F. 跨城市迁移预测（研究型）

**为什么选这个？**
- 体现迁移学习思想
- 三城市地理位置接近但气候差异明显（敦煌高原、金昌戈壁、酒泉盆地）
- 可探索领域自适应（Domain Adaptation）方法

**建模方案**：
1. **源域训练**：用敦煌数据训练模型
2. **目标域微调**：在金昌/酒泉数据上微调最后几层
3. **对比实验**：直接训练 vs 迁移学习 vs 联合训练

**评估指标**：与单城市预测相同的指标，额外对比迁移前后的提升

**预期难度**：高。需要理解领域偏移问题。

## 三、推荐方案

### 方案一：保守稳妥（适合时间有限）

**选择方向**：A（温度单步预测）或 E（极端温度预警）

**理由**：
- 数据质量高，无需大量清洗
- 模型简单，调参空间小
- 2-3 小时可完成完整流程
- 结果稳定，不容易"翻车"

### 方案二：平衡深度（推荐）

**选择方向**：B（温度多变量预测）或 C（降水二分类）

**理由**：
- 体现完整的 ML 流程：特征工程 → 模型选择 → 调参 → 评估
- 可对比多个模型（XGBoost vs LSTM），体现分析能力
- 有实际意义，报告撰写素材丰富

### 方案三：挑战进阶（适合有基础）

**选择方向**：F（跨城市迁移）

**理由**：
- 体现迁移学习这一重要 ML 概念
- 可做消融实验（对比单城市/迁移/联合训练）
- 报告有亮点，容易获得高分

## 五、关键注意事项

### 5.1 时间序列切分（重要！）

**禁止**：随机打乱后划分训练/测试集！这会导致数据泄露（未来信息泄露到过去）。

**正确做法**：
```
训练集：2005-01 ~ 2020-12（约 80%）
验证集：2021-01 ~ 2022-12（约 10%）
测试集：2023-01 ~ 2025-12（约 10%）
```

### 5.2 缺失值处理

- **T, U, Ff**：缺失极少，直接前向填充或删除
- **RRR（降水量）**：缺失即无记录，需与 "无降水" 区分处理
- **ff10, sss, Tg**：缺失极多，直接丢弃这些列

### 5.3 归一化/标准化

- 数值特征（T, U, Ff 等）必须标准化后再输入神经网络
- 树模型（XGBoost, RandomForest）对归一化不敏感

### 5.4 类别不均衡

- 如果做降水预测，正负样本比例约 56:44，轻微不均衡
- 如果做天气现象多分类，严重不均衡，需用 class_weight 或采样策略
