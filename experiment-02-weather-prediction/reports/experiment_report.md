## 一、实验目的

1. 掌握时间序列预测任务的完整流程：数据探索、特征工程、时序切分、模型训练与评估。
2. 学习并对比传统机器学习（XGBoost）与深度学习（LSTM）在多变量时序回归任务上的性能差异。
3. 理解滞后特征、周期性编码等时序特征工程方法对预测效果的贡献。
4. 通过实际气象数据，分析温度序列的自回归特性与周期规律。

| 决策项 | 选择 |

### 4.2 数据切分策略


**关键约束**：时间序列预测必须按时间顺序切分。若随机打乱，会出现"用未来预测过去"的数据泄露。

## 五、特征工程

### 5.1 时间周期性编码

气象数据具有强周期特征。用正弦/余弦编码避免数值断裂：

```
hour_sin  = sin(2π · hour / 24)
hour_cos  = cos(2π · hour / 24)
month_sin = sin(2π · month / 12)
month_cos = cos(2π · month / 12)
dayofyear_sin = sin(2π · dayofyear / 365)
dayofyear_cos = cos(2π · dayofyear / 365)
```

**作用**：将 23:00 与 00:00 的"周期性距离"从 23 缩小到 1，使模型正确理解日周期。

### 5.2 滞后特征（Lag Features）

| 滞后步 | 物理含义 |
|--------|----------|
| lag 1 | 3 小时前 |
| lag 2 | 6 小时前 |
| lag 3 | 9 小时前 |
| lag 4 | 12 小时前 |
| lag 8 | 24 小时前 |
| lag 12 | 36 小时前 |
| lag 24 | 72 小时前 |

### 5.3 滑动窗口统计

- `T_rolling_mean_8`：过去 24 小时（8 个 3h 窗口）的平均温度
- `T_rolling_std_8`：过去 24 小时温度标准差（反映温度波动剧烈程度）

### 5.4 特征总数

经上述工程后，共构造 **33 维数值特征**。

## 六、模型架构

### 6.1 XGBoost（梯度提升树）

- **类型**：集成学习，串行训练 500 棵 CART 回归树
- **正则化**：`subsample=0.8` + `colsample_bytree=0.8`，每棵树仅见部分样本和特征
- **损失函数**：`reg:squarederror`（均方误差）
- **优势**：对表格型特征天然适配，训练快，可解释性强

### 6.2 LSTM（长短期记忆网络）

```
Input (batch, seq_len=24, features=33)
    |
LSTM Layer 1 (hidden=64, dropout=0.2)
    |
LSTM Layer 2 (hidden=64, dropout=0.2)
    | 取最后一个时间步隐藏状态
    v
FC (64 -> 32) + ReLU + Dropout(0.2)
    v
FC (32 -> 1)
    v
Output: 预测温度值
```

- **优势**：天然建模序列依赖，理论上可捕获长期时序模式
- **劣势**：当前架构相对轻量（hidden=64，2 层），在强自回归特征面前优势不明显

## 七、实验结果

### 7.1 核心指标对比

| 指标 | XGBoost | LSTM | 优势方 |
|------|---------|------|--------|
| **Test R²** | **0.9949** | 0.9224 | XGBoost (+0.0725) |
| Test RMSE | **0.8889** | 3.4772 | XGBoost (低 3.9×) |
| Test MAE | **0.6630** | 2.8168 | XGBoost (低 4.2×) |
| Train R² | 0.9977 | — | — |
| Val R² | 0.9947 | 0.9386 | XGBoost (+0.0561) |
| 训练耗时 | ~3 分钟 | ~10–15 分钟 | XGBoost 快 3–5× |
| 早停 Epoch | 未触发 (499/500) | Epoch 25/50 | — |

### 7.2 模型性能诊断

**XGBoost**：
- Train R² (0.9977) 与 Test R² (0.9949) 差距仅 **0.0028**，说明正则化生效，无过拟合。
- Best iteration = 499，早停未触发，500 棵树尚未完全收敛，继续增加可能有微弱提升。
- 平均预测误差约 **0.89°C**（RMSE），在实际气象预报中属于高精度水平。

**LSTM**：
- Val R² 在 Epoch 8 达到峰值 0.9328，之后震荡下降，Epoch 21 后不再提升。
- 早停于 Epoch 25，说明模型已收敛到局部最优。
- Test R² = 0.9224 也是强基线，但显著低于 XGBoost。

### 7.3 特征重要性分析（XGBoost Top 10）

| 排名 | 特征 | 重要性 | 物理含义 |
|------|------|--------|----------|
| 1 | `T_lag8` | 31197.8 | **24 小时前的温度** — 日周期主导 |
| 2 | `T_lag1` | 14063.0 | **3 小时前的温度** — 短时自回归 |
| 3 | `Tx` | 2664.1 | 当日最高气温 |
| 4 | `hour_cos` | 2613.9 | 日周期余弦编码 |
| 5 | `T_lag24` | 2163.2 | **72 小时前的温度** |
| 6 | `Tn` | 1914.9 | 当日最低气温 |
| 7 | `hour_sin` | 1549.7 | 日周期正弦编码 |
| 8 | `hour` | 907.3 | 小时数值 |
| 9 | `P` | 539.8 | 海平面气压 |
| 10 | `U` | 421.2 | 相对湿度 |

**关键发现**：
- **历史温度自身是核心预测信号**：`T_lag1`、`T_lag8`、`T_lag24` 合计占绝对主导，说明温度序列具有极强的自回归特性。
- **日周期特征次之**：`hour_cos`、`hour_sin`、`hour` 反映了温度随昼夜变化的规律性。
- **气象参数贡献有限**：气压 `P`、湿度 `U`、风速 `Ff` 虽有一定贡献，但远小于历史温度本身。

### 7.4 误差分布分析

XGBoost Test RMSE = 0.89°C，意味着约 68% 的预测误差在 ±0.9°C 以内。考虑到三城市跨度 20 年、温度范围约 -35°C ~ +25°C，该精度说明模型已成功捕捉到：
1. 日周期（昼夜温差）
2. 年周期（季节变化）
3. 短时自回归（相邻时刻温度连续性）

## 八、结果分析与讨论

### 8.1 为什么 XGBoost 显著优于 LSTM？

#### 原因一：特征与模型的天然匹配

预处理后得到的 33 维特征全是**结构化表格数据**（滞后值、滑动统计、正弦/余弦编码）。XGBoost 对这类特征的处理效率天然高于 LSTM：
- 树模型通过特征分裂直接捕捉 `T_lag8`（24h 前温度）与当前温度的高度相关性。
- LSTM 需要将 33 维特征沿 24 个时间步展开为 `(24, 33)` 的张量，其中大部分信息变化微弱，增加了学习难度。

#### 原因二：数据规模与模型容量的匹配

- 训练集 13 万条，对 XGBoost 是甜点区：足够训练 500 棵深度为 6 的树，且不会严重过拟合。
- 对 LSTM 来说，13 万条序列样本不算大，而当前架构（hidden=64，2 层）参数量仅约 5 万，容量偏保守。

#### 原因三：任务本身的自回归特性

温度预测的核心信息来自"历史温度本身"，而非复杂的非线性时序模式。对于这种强自回归、弱长程依赖的任务，树模型的特征分裂机制比循环神经网络更高效。

### 8.2 LSTM 的潜在改进方向

若坚持优化 LSTM，以下调整可能缩小差距：

| 改进方向 | 具体做法 | 预期效果 |
|----------|----------|----------|
| 增大容量 | hidden=128, num_layers=3 | 提升表达能力，可能 +0.01–0.02 R² |
| 调整序列长度 | seq_len=8（24h）或 seq_len=48（144h） | 24 步可能包含冗余，8 步更聚焦 |
| 仅输入关键特征 | 只保留 T 序列 + hour_sin/cos | 降低噪声，简化学习 |
| 学习率调优 | 使用 CosineAnnealing 替代 ReduceLROnPlateau | 更稳定的收敛 |
| 模型替换 | GRU 或 Temporal Fusion Transformer | 可能捕获更复杂的时序交互 |

但保守估计，即使充分调优，LSTM 的 Test R² 很难突破 0.95，与 XGBoost 的 0.9949 仍有差距。

### 8.3 特征工程的验证

特征重要性排名验证了工程设计的有效性：
- `T_lag8`（24h 前）> `T_lag1`（3h 前）> `T_lag24`（72h 前），说明**日周期是首要规律**，短时连续性次之。
- `hour_cos` / `hour_sin` 排名 4/7，证明正弦/余弦编码成功让模型理解了日周期。
- `T_rolling_mean_8`（24h 滑动平均）排名 12，虽非 Top 但仍有贡献，说明平滑历史信息有辅助价值。

### 8.4 已知问题与局限

1. **MAPE 计算异常**：测试集中存在 T = 0°C 的样本，MAPE 公式 `|y_true - y_pred| / y_true` 除以零导致 `inf`。不影响 RMSE/MAE/R² 的有效性。
2. **未做消融实验**：未逐一移除滞后特征、周期编码等验证单项贡献。
3. **未考虑空间差异**：三城市气候差异明显（敦煌高原、金昌戈壁、酒泉盆地），当前合并训练未区分城市特异性。
4. **LSTM 未标准化输入**：脚本中缺少对特征的 Z-Score 标准化，可能对 LSTM 收敛有一定影响。

## 九、实验总结

### 9.1 主要结论

本实验基于甘肃省敦煌、金昌、酒泉三市约 20 年的 3 小时间隔气象数据，完成了**未来 3 小时气温预测**的回归任务。通过构造 33 维时序特征（滞后值、滑动统计、周期编码），分别训练了 XGBoost 和 LSTM 两个模型。

**核心结论**：
1. **XGBoost 为最优方案**，Test R² = **0.9949**，RMSE = **0.89°C**，在 3 小时间隔的温度预测上达到高精度水平。
2. **温度序列的自回归特性是预测的核心驱动力**：24 小时前的历史温度（`T_lag8`）是最重要的特征，贡献远超湿度、气压等辅助气象参数。
3. **结构化时序特征更适合树模型**：在强自回归、弱长程依赖的任务上，XGBoost 对表格型滞后特征的处理效率显著优于 LSTM。

### 9.2 遇到的问题与解决方案

| 问题 | 解决方案 |
|------|----------|
| Pandas 2.x 移除了 `fillna(method='ffill')` | 改用 `.ffill()` / `.bfill()` |
| 预处理时 `dropna()` 删除了 99.9% 的数据 | 对**所有列**（包括 object 类型）做缺失值填充，并删除缺失率 >95% 的列 |
| XGBoost 3.2.0 的 sklearn API 不支持 `early_stopping_rounds` | 迁移到原生 `xgb.train()` + `DMatrix` + `Booster` API |
| LSTM 收敛后 Val RMSE 震荡 | `ReduceLROnPlateau` + EarlyStopping (patience=10) 自动终止训练 |

### 9.3 收获与反思

**技术收获**：
- 时间序列预测的核心是**特征工程**：滞后特征和周期编码的质量直接决定模型上限。
- **模型选择需匹配数据结构**：表格型特征优先树模型，原始序列信号优先神经网络，没有万能模型。
- 时间序列切分必须严格遵守**时序顺序**，随机打乱是隐性数据泄露。

**工程反思**：
- 第三方库版本兼容性（XGBoost 3.x API 大改）是部署中的常见陷阱，应在 `requirements.txt` 中锁定确切版本。
- 缺失值处理策略需覆盖全部列类型，不能仅针对 `numeric` 列。
- 实验报告中应包含**消融实验**或**特征重要性分析**，以验证工程设计的有效性。
